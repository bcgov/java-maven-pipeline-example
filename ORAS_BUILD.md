# ORAS local build & test instructions

This document describes how to locally build the Maven artifact, package it as a single tarball suitable for ORAS, and how to preview or test ORAS operations without an automatic registry push.

Prerequisites
- zsh (or bash)
- ./mvnw present in repo root
- `env.sh` and `VERSION` file (generated by composer)
- `jq` (optional but recommended) for JSON generation
- `oras` binary (optional for pull/push tests)

Quick workflow (zsh-friendly)
1. From the repo root:

```bash
cd /home/andrwils/projects/bcgov/java-maven-pipeline-example
set -euo pipefail

# Load CI environment (reads VERSION) without contacting Vault
source ./env.sh build ./ --skip-vault

# Build the package (skip tests)
./mvnw --batch-mode -Dmaven.test.skip=true clean package

# Resolve coordinates
ARTIFACT_ID=$(mvn help:evaluate -Dexpression=project.artifactId -q -DforceStdout --file ./pom.xml)
PACKAGING=$(mvn help:evaluate -Dexpression=project.packaging -q -DforceStdout --file ./pom.xml)
PROJECT_VERSION="$VERSION"

# Verify artifact exists and create tarball for ORAS
ARTIFACT_FILE="target/${ARTIFACT_ID}-${PROJECT_VERSION}.${PACKAGING}"
ls -l "$ARTIFACT_FILE"
TAR_NAME="${ARTIFACT_ID}-${PROJECT_VERSION}.tar.gz"
tar -C target -czf "$TAR_NAME" "$(basename "$ARTIFACT_FILE")"

# Inspect tarball
ls -lh "$TAR_NAME"
tar -tzf "$TAR_NAME"
sha256sum "$TAR_NAME"
stat -c%s "$TAR_NAME"   # file size in bytes
```

2. Create `annotation.json` (recommended; uses `jq` to avoid quoting issues):

```bash
REPO_URL="https://github.com/bcgov/java-maven-pipeline-example"
jq -n --arg id "$ARTIFACT_ID" --arg v "$PROJECT_VERSION" --arg src "$REPO_URL" '{
  "$manifest": {
    "org.opencontainers.image.description": ("Build artifact for " + $id + " version " + $v),
    "org.opencontainers.image.licenses": "Apache-2.0",
    "org.opencontainers.image.source": $src,
    "org.opencontainers.image.title": ($id + "-artifact"),
    "org.opencontainers.image.version": $v
  }
}' > annotation.json
```

3. Prepare credentials (zsh prompt style):

```bash
read "?Enter your GitHub user: " GHCR_USER
read -s "?Enter your GitHub Token: " GHCR_TOKEN
echo

# Login (this will authenticate to GHCR if token is valid)
printf "%s" "$GHCR_TOKEN" | oras login ghcr.io -u "$GHCR_USER" --password-stdin

# After use, unset the token
unset GHCR_TOKEN
```

4. Preview the ORAS push command (no automatic push):

```bash
ORAS_REPO="ghcr.io/bcgov/java-maven-pipeline-example"
PROJECT_TAG="$PROJECT_VERSION"
mkdir -p artifacts
MANIFEST_FILE="artifacts/${ARTIFACT_ID}-${PROJECT_VERSION}-manifest.json"
echo oras push --annotation-file annotation.json --export-manifest "${MANIFEST_FILE}" ${ORAS_REPO}/package:${PROJECT_TAG} ${TAR_NAME}

# To actually push, remove the 'echo' or run the printed command (requires write access).
```

5. Pulling an existing package (use same variables):

```bash
# login as above, then
mkdir -p artifact-out
oras pull --output ./artifact-out ${ORAS_REPO}/package:${PROJECT_TAG}
ls -la artifact-out
tar -tzf "artifact-out/${TAR_NAME}"
sha256sum "artifact-out/${TAR_NAME}"
stat -c%s "artifact-out/${TAR_NAME}"
```
